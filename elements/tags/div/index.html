<div>

A block level element with no semantic meaning at all.

Overview

For lack of a better analogy, the <div> are the blue jeans of the HTML world. They are incredibly versatile, nobody looks down on you to wearing/using them, and quite often they are the only thing that gets the job done.

The W3C discourages their use:

Authors are strongly encouraged to view the div element as an element of last resort, for when no other element is suitable. Use of more appropriate elements instead of the div element leads to better accessibility for readers and easier maintainability for authors.

I'd say don't take too much stock in that. It's a bit like environmental groups saying We discourage the use of gasoline. When available, use alternatives in fueling you car. It's true, but the cases where you cannot use the reccomended solution outnumber those where you can. So don't let that drive you to picking obscure elements to avoid a <div>. Just make sure you think about it a moment before you do.

Conflicting or Confusing Elements

<p>

The <p> element gets confused for a generic element from time to time. Especially since before people cared about semantics, it was used that way. It's simple, if it is a paragraph, then use a <p>, otherwise, you can use a <div>.

<span>

A <span> is a generic inline element, not block level.
