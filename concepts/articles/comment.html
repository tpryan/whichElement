Comment

An audience provided response to a piece of content on the web.

This entry refers to an indivdual comment on a blog. For the list that contains this item see comments

Overview

When we refer to comments, typically requires 2 things:

  1. A response to the main content
  2. An identifier for the author of the response.

Then there are optional pieces of content:

  • A website for the author of the response.
  • Timestamp for the response.

Candidates

  • <article>
  • <div>
  • <p>
  • <span>

Right off the bat, I think we can nix the <p>. The <p> should not contain other block-elements which precludes it from containing the members of a comment. Which makes you have to use a lot of custom CSS to get the tags you can use in a <p> layout the appropriate way for a comment.

Much the same way, this precludes the <span> as it cannot contain block-elements either, which makes for a lot of CSS hoop jumping.

Up until recently, the <div> would be your goto tag for a comment. As it can contain block-level elements, can be styled without too much drama, and is specifically for holding information that isn't semantically designated any other way. It's a good choice.

The <article> is also a great candidate. But, it denotes standalone content. At first glace this would seem to preclude using it. However, comments are often deep linked. This makes it seem like there is value in directly going to a comment and bypassing the original content. In this case, the comment can be looked upon as standalone and deserving of an <article>.

Verdict

We recomend the <article>. The <div> can also work, but the <article> is the better semantic choice.
